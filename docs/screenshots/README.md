## Capture Guidance

- Use a real installed build when possible, not a dev window.
- Capture at about 1400px wide after retina crop.
- Keep the app window light enough for GitHub to load quickly.
- Prefer realistic sample data over empty states.
- Show the polished workflow, not debug tools or terminal windows.
- Avoid browser tabs, desktop files, or menu bar items that distract from the app.

## Privacy Checklist

Before committing screenshots, inspect each image for:

- API keys, tokens, email addresses, phone numbers, and private domains.
- Customer names, patient names, employee names, or private chat content.
- Internal IPs, database names, ticket IDs, and local filesystem paths.
- Browser tabs or window titles unrelated to the screenshot.

Use generic demo data when in doubt.
